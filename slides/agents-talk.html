03 / The Agent World

Agents do do do.

Read the file.
Run the tests.
Tell me when it's done

It's a while loop.

while True:
    response = llm(messages, tools)
    if response.stop_reason != "tool_use":
        break  # done
    result = execute(response.tool_call)
    messages.append(result)  # loop continues

04 / Tools

A loop with no capabilities is useless.

Every computer has the most powerful tool. You use it every day.

The terminal.

A tool is a function.

Clear inputs. Clear outputs. Guardrails.

def run_bash(command):
  # guardrails
  dangerous = ["rm -rf /", "sudo", "shutdown"]
  if any(d in command for d in dangerous):
    return "Error: blocked"
  # clear input → clear output
  result = subprocess.run(command, shell=True)
  return result.stdout

Inside execute().

Skills vs Tools

Tools

Executable functions. When called, something happens: a file gets written, an API gets hit.

The hands that do things.

Skills

Packaged expertise. Context, instructions, domain knowledge that makes the agent better.

The training that knows things.

08 / OpenClaw

How to scale?

Parallel agents. More skills. More channels.

OpenClaw =

Agent Loop (what we just learned)
+ Gateway (routes WhatsApp, Slack, Discord...)
+ Memory (remembers across sessions)

The Gateway.

A WebSocket server.

Memory.

The agent remembers across sessions.

Each agent has its own isolated memory. No sharing between agents.

The full picture.

And speaking about injecting files...

Every session loads these from the workspace:

~/.openclaw/workspace/

AGENTS.md    # rules, priorities, how to behave
SOUL.md     # persona, tone, boundaries
USER.md     # who you are
TOOLS.md    # local tool conventions
IDENTITY.md # agent name, vibe
MEMORY.md   # long-term curated facts
memory/     # daily logs (YYYY-MM-DD.md)
skills/     # workspace-specific skills

Plain markdown. You can edit them yourself.
